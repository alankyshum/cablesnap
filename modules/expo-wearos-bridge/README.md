# expo-wearos-bridge

App-local Expo native module + Wear OS companion Gradle template for CableSnap.

This package is **the M0 (build-infrastructure) skeleton** of the Wear OS pipeline
defined in [`.plans/PLAN-BLD-716.md`](../../.plans/PLAN-BLD-716.md). M0 is a
no-behavior-change milestone: the JS API exported here is a no-op stub, and the
native Kotlin/Compose code is empty — just enough for the Gradle modules to
compile and the F-Droid flavor split to be enforced.

Real bridge behavior (`MessageClient`/`DataClient`, watch UI screens, durable
event queue) lands in M1..M5.

## Layout

```
modules/expo-wearos-bridge/
  expo-module.config.json
  android/                                # phone-side bridge (Expo native module)
    src/main/java/com/persoack/cablesnap/wearbridge/
      WearOSModule.kt
  ios/                                    # iOS has no Wear OS analog — empty stub
  src/                                    # JS API surface
    index.ts
    types.ts
  wear-template/                          # standalone Wear OS Gradle module
                                          # copied into android/wear/ by the
                                          # plugins/with-wearos-module Config
                                          # Plugin during `expo prebuild`.
    build.gradle.kts
    src/main/AndroidManifest.xml
    src/main/java/com/persoack/cablesnap/wear/MainActivity.kt
```

## Why this structure

CableSnap is a **managed Expo** app — `android/` is generated by `expo prebuild`,
not checked in. To get a Gradle `:wear` subproject into the prebuild output we
ship the watch APK source in this package's `wear-template/` directory and copy
it during prebuild via `plugins/with-wearos-module.js`. Decision rationale:
PLAN-BLD-716.md §"Repo integration strategy (decided per TL-1)".

## F-Droid flavor purity (AC10b)

The F-Droid build (`assembleFdroidRelease`) **must not** include any
`com.google.android.gms.wearable` classes. This is enforced two ways:

1. The Config Plugin places the GMS dependency inside `playReleaseImplementation`
   only (never in `implementation` or `fdroidReleaseImplementation`).
2. CI runs `unzip -l app-fdroidRelease.apk | grep -c 'com/google/android/gms/wearable'`
   and fails the build if the count is non-zero.

See PLAN-BLD-716.md §"F-Droid + Play split (decided per TL-2)".
